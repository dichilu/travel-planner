# 更新日誌 (Changelog)

所有關於「行程規劃大師」的版本更新、功能新增與 Bug 修復紀錄將統一維護於此。

## [v1.4.0] - 最新更新 (Current)

### 🚀 新增與改進 (Added & Improved)
- **YouTube 字幕引擎正式對接 RapidAPI 專業服務**：
  - **突破機房封鎖**：徹底解決 Vercel 等雲端伺服器機房 IP 遭 YouTube 封鎖的問題。改為呼叫 RapidAPI 上的 `youtube-transcript3` 專業數據服務，藉由其龐大的住宅 IP 代理網路，實現 100% 穩定抓取。
  - **安全性提升**：API Key 妥善封裝在後端的 Vercel Serverless Function 中，保障金鑰安全不外洩。
  - **安全免費額度**：採用 Hard Limit 機制，每月享有 100 次完全免費的穩定額度，用罄自動停止，絕無意外扣款風險。
- **導入「體力遞減原則 (Energy Tapering Engine)」**：為了讓旅遊體驗更符合真實人性，AI 規劃邏輯現在會將「遠程/高消耗體力」的活動優先壓縮在旅途前半段，後半段則自動過渡至「悠閒/定點」模式。
- **修正每日結尾邏輯 (Day-Ending Mandate)**：修正行程有時會結束在晚餐餐廳的問題，強制 AI 在每天結尾加上「返回指定飯店」的防呆交通規劃。

### 🐛 修正 (Fixed)
- **長行程 PDF 匯出全白修復**：解決了因行程過長導致 `html2canvas` 繪製超出瀏覽器極限而產生全白 PDF 的問題。現已加入高度防呆機制（大於 15000px 自動切換至瀏覽器內建列印）。
- **PDF 列印背景消失修復**：在全域 CSS 中加入了 `-webkit-print-color-adjust: exact`，強制瀏覽器在列印時保留所有漸層與背景顏色，確保 PDF 質感不打折。
- **手機版面重疊修復**：解決了「自選航班編號」與「推薦航班」區塊在手機小螢幕上，因為 CSS 寫死高度 (`h-[340px]`) 導致文字換行、內容溢出而重疊的排版錯誤。已全面改為自適應高度 `h-auto md:h-[340px]`。

## [v1.3.0]

### 🐛 修正 (Fixed)
- **YouTube 字幕擷取全面失效修復 (Transcript Fetch Rewrite)**：
  - **根本原因**：原先使用的 `youtube-transcript` 第三方套件在 Vercel 的雲端伺服器 IP 上被 YouTube 反爬蟲機制全面封鎖，導致所有影片的字幕擷取均回報「無 CC 字幕」而靜默失敗。
  - **解決方案**：完全移除 `youtube-transcript` 套件依賴，改為自訂的零依賴字幕擷取引擎 (`api/getTranscript.js`)，使用原生 `fetch` 搭配三重 fallback 策略：
    1. **Android InnerTube API**（v20.10.38 用戶端偽裝，成功率最高）
    2. **Web InnerTube API**（加入 GDPR consent cookie 繞過歐洲同意頁面）
    3. **YouTube 網頁爬取**（加入 consent cookie 的 HTML 解析 fallback）
  - **中文字幕優先選取**：新增自動偵測邏輯，優先選取 `zh-Hant` → `zh-TW` → `zh` → 任意語言的字幕軌道，確保中文影片的字幕能正確被擷取。
  - **前端零改動**：API 的輸入輸出格式維持不變，前端 `App.jsx` 無需任何修改。

## [v1.2.0]

### ✨ 新增 (Added)
- **YouTube 網紅行程自動萃取與整合**：新增 Vercel Serverless Function 支援。使用者只需貼上 YouTube Vlog 網址，系統會自動抓取影片逐字稿，並啟動「三角過濾器 (Veto Protocol)」，將影片精華去蕪存菁地完美融入使用者的客製化行程中。
- **餐廳主從反轉邏輯**：當使用者選擇「美食狂熱」風格時，AI 將打破「就近吃飯」的原則，改為以「高評價 / 網紅名店」為核心目的地，圍繞餐廳規劃行程。
- **最終行程表即時編輯與連動推移 (Inline Editing & Auto Shift)**：為了提升最終匯出 PDF 的專業度與準確性，在最後的行程表畫面 (Step 3) 的「Confirmed Flight」與「Confirmed Hotel」區塊新增了「編輯按鈕 (✏️)」。特別是在航班編輯中，新增了結構化的「落地時間選擇器」。當使用者輸入實際落地時間並儲存後，系統不僅會完美覆寫 PDF 頂部的文字，更會觸發「骨牌推延引擎」，自動將第一天所有的後續行程時間進行精準的數學推移，徹底解決手動改時間的煩惱。
- **預算階層與網紅推薦整合**：修復前端未將預算變數傳遞給 AI 的 Bug。現在 AI 會嚴格根據選擇的預算（背包客、標準、輕奢、無上限）提供對應的推薦，並在各級別中融入專屬的「網紅打卡點」元素。
- **物流排版視覺等高對齊 (Cross-Column Equal Height)**：將步驟二 (機票) 與步驟三 (飯店) 的「自填卡片」與「系統推薦卡片」強制採用 CSS Grid 與固定高度進行完美對齊，徹底消除選項間忽長忽短的視覺不平衡，提升整體 UI 專業感。
- **航班自選欄位精細化**：重構「自選機票」的輸入邏輯，從單一字串擴充為「航班代號、起飛時間、落地時間」的獨立輸入，並結合 Grid 三等份排版防止寬度互相擠壓。
- **住宿自選欄位擴充**：將「自選飯店」分離為「飯店名稱」與「所在城市」雙輸入框，避免 AI 定位錯誤。

### 🐛 修正 (Fixed)
- **YouTube 字幕擷取防呆與 ESM 模組修正**：
  - 修復 `youtube-transcript` 套件在 Vercel 環境中的 ES Module 匯入崩潰問題。
  - **優化使用者體驗 (Smart Alert & Warn-and-Continue)**：若使用者提供多支影片，系統會自動辨識並回報「成功載入的數量」與「因無 CC 字幕而被忽略的影片網址」。這能避免「靜默失效 (Silent Failure)」造成的 AI 信任危機，並在提示後順暢地繼續生成行程。
- **Google Flights 連結失效修正**：修復當目的地包含斜線（例如「大阪/京都」）時，會導致 Google Flights 自然語言解析器崩潰而無法帶入日期與地點的問題。已在傳遞參數前增加正規表達式進行字串清洗（僅取「大阪」）。
- **自選航班驗證 Bug 修正**：修復當使用者填寫了「自選飯店」但未填寫「自選航班」時，AI 預先驗證機制會誤將「空字串」判定為無效資料而阻擋行程產生的問題。現已明確指示 AI 空字串為合法預設值。
- **邊界時間物理引擎升級 (Boundary Events)**：
  - 修復「出門前往機場」的時間模糊問題，強制要求 AI 精準將第一站設定為起飛前 3 小時出發。
  - 新增「降落通關緩衝」，強制 AI 在落地後必須預留 1 至 1.5 小時的海關通關與提領行李時間，才可開始安排後續交通，確保行程時間軸符合現實物理限制。

## [v1.1.0] - 2026-04-23

### ✨ 新增 (Added)
- **Klook / KKday 動態找票功能**：當行程被判定為需要門票或車票時（`ticketRequired: true`），AI 會自動萃取精準的專屬搜尋關鍵字 `ticketSearchKeyword`（例如：Kansai Airport Haruka），並在畫面中提供一鍵前往 Klook 或 KKday 搜尋的按鈕。
- **交通找票支援**：解除原本只在「景點」顯示找票按鈕的限制，現在只要需要買票的交通票券（如 JR Pass、機場快線），也支援一鍵精準搜尋。
- **AI 預先驗證機制**：在行程產生前，增加對使用者自填航班或飯店名稱的合理性檢查。

### 🐛 修正 (Fixed)
- **預算分析幣值錯亂修正**：修復 AI 預算估算時錯用語系幣值的問題，現已強制 AI 必須以「出發地當地貨幣（如台幣 TWD）」進行預估，讓畫面的加總金額顯示符合常理。
- **UI 畫面崩潰 Bug 修正**：修復先前因拆分「去程航班」與「回程航班」，導致 UI 遺留舊變數 `customFlight` 造成的白畫面錯誤。
- **API 連線錯誤處理**：增加對 429 (Rate Limit Exceeded) 與 403 (Forbidden) 錯誤的防護機制與白話文錯誤提示。
